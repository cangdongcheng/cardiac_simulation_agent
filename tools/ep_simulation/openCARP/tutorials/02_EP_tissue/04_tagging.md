# CLAUDE.md — 02_EP_tissue / 04_tagging

## What this tutorial demonstrates

How to assign **region tags** to mesh elements in openCARP via two mechanisms:

1. **Mesher-level tagging** — defined when the mesh is generated (`regdef[*].*` in mesher). Types: sphere (type=1), block (type=0), cylinder (type=2). Units: **cm** (not µm). First-matching-region rule by default.
2. **Dynamic retagging** — applied by openCARP at simulation startup (`tagreg[*].*`). Types: sphere (type=1), block (type=2), cylinder (type=3), elemfile (type=4). Units: **µm**. Highest-tag-wins rule.

This tutorial runs `-experiment 3` (model build only — no time stepping). It generates and tags a mesh then writes out the result as `retagged.dat`.

## CLI

```bash
cd /usr/local/lib/opencarp/share/tutorials/02_EP_tissue/04_tagging

# Default: no special regions (just bath + tissue tag 0)
PATH=... python run.py --np 1

# Mesher-defined sphere inside block + all three dynamic types
PATH=... python run.py --mesher sphere block steps --dynamic sphere cylinder block --np 1

# Mesher sphere as bath region
PATH=... python run.py --mesher sphere --add-bath --np 1
```

`--mesher` and `--dynamic` accept multiple values (space-separated), and the order matters.

## Tag scheme

| Region | Mechanism | Type arg | Tag assigned | Notes |
| --- | --- | --- | --- | --- |
| Bath (default) | mesher | — | 0 | Elements outside all explicit regions |
| sphere | mesher | sphere | 10 (1st), 20 (2nd), … | 10 × (1-indexed position in `--mesher` list) |
| block | mesher | block | 20 | Half-cube: x∈[−x/2,0] |
| steps | mesher | steps | 30, 31, 32, 33 | 4 equal z-slabs |
| dynamic sphere | dynamic (openCARP) | sphere (type=1) | 100 (1st), 200 (2nd), … | At (0, 700, 0) µm, r=500 µm |
| dynamic cylinder | dynamic | cylinder (type=3) | 200 | Axis along x through (x, 2000, 1500) µm, r=250 µm |
| dynamic block | dynamic | block (type=2) | 300 | x∈[−2500,1000], y∈[−1000,1000], z∈[−200,100] µm |

- `retagged.dat` not generated (no dynamic retagging, openCARP skips writing it)

- 1,715,000 total elements (5 mm cube with bath at 100 µm resolution)
- Tag distribution: 0 (bath: majority), 10 (sphere: 15,062), 20 (block: 299,700), 30–33 (steps: ~301,140), 100 (dyn sphere: 1,658), 200 (dyn cylinder: 1,440), 300 (dyn block: 6,000)
- Dynamic tags always win over mesher tags when they overlap (highest-tag-wins rule)

## Key rules to remember

1. **Unit inconsistency**: mesher uses **cm** for coordinates; dynamic retagging (`tagreg[*].*`) uses **µm**. This is a common source of confusion.
2. **Conflict resolution**: mesher → first-matching-region wins (by default; `--first_reg False` inverts this); dynamic → highest tag wins.
3. **If a tag is not in any region, it becomes region 0.** Region 0 gets the properties of `imp_region[0]` and `gregion[0]` — important if you have stray bath elements inside a simulation.
4. **Dynamic tags must be registered in `IntraTags`/`ExtraTags`** even though they're generated by openCARP. Failure to register them means those elements are excluded from physics regions.
5. **Mesher bath elements cannot be turned into myocardium** by dynamic retagging (the reverse is possible).
6. **`-experiment 3`** runs model build only — it sets up the physics solver (including dynamic tagging) but does not time-step. Useful for validating mesh/tag setup before running a full simulation.

## Lessons (transferable)

1. Use `-experiment 3` in `--dry` mode to validate that your tagging scheme produces the expected tag distribution before running a full simulation.
2. The `retagged.dat` file (one element tag per line) can be visualized in meshalyzer as an element-data field to inspect the tag geometry.
3. Dynamic cylinder definition: `p0` is a point on the cylinder axis, `p1` is another point on the axis (direction vector). The `radius` is the cylinder radius in µm. Length is not specified — the cylinder extends infinitely.
4. `block_e.elem` contains only the extracellular (bath + tissue) elements; `block_i.elem` contains only intracellular (tissue) elements. This bidomain split is controlled by `IntraTags` and `ExtraTags` passed to `gen_physics_opts()`.
